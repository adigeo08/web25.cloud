☁️ What is Web25.Cloud?

                        

                            Web25.Cloud is a decentralized web platform built on top of
                            PeerWeb. It extends peer-to-peer website
                            hosting with a local in-browser signing wallet,
                            signed torrent deployments, P2P direct messaging, and browser-native
                            distribution without a central server.
                        

                    


                    

                        
💬 P2P Direct Messenger

                        
Exchange offer/answer signaling codes manually, then chat directly over WebRTC data channels with end-to-end asymmetric encryption tied to your EVM identity.

                        

                            
🔐 ECIES secp256k1 Encryption — Each message encrypted with recipient's secp256k1 public key (EVM curve)

                            
✍️ Per-message Signatures — Every message signed with sender's private key and verified on receipt

                            
🪪 Built-in Identity Verification — Public key → keccak256 → address verified at connect time

                            
🧭 Manual Signaling UX — Host offer + guest answer exchange with identity embedded

                            
🧊 STUN Discovery — Uses Google's public STUN for ICE candidates

                            
🛰️ Pure P2P Transport — No centralized chat backend required

                        

                        


                        
🔏 Signed Torrent Publishing

                        
Publish artifacts are signed in browser and embedded directly in torrent metadata.

                        

                            
🧾 Deterministic Payload — Reproducible fields before signing

                            
✍️ Local Signature — Signing key never leaves your browser

                            
🧷 Embedded Proof — Publisher, signature, algorithm, and timestamp in .torrent

                            
✅ Verification on Load — Signature status shown when a site is opened by hash

                        

                        


                        
🪪 Local Identity + WebAuthn Passkey Protection

                        

                            Your EVM signing identity lives entirely in your browser,
                            protected by your device's biometric authenticator via the WebAuthn standard.
                        

                        

                            
🔐 WebAuthn Passkey — Private key protected by FaceID / TouchID / device PIN

                            
🧬 Zero Server — Identity and keys never leave your device

                            
🪪 EVM Compatible — secp256k1 keypair, standard Ethereum address

                            
🌱 Seed Phrase Recovery — BIP-39 mnemonic for cross-device restore

                            
➕ Multi-Passkey — Add FaceID + TouchID as alternates on same wallet

                            
⏱️ Session Cache — Biometric session cached 30 min, auto-expires

                            
🔒 Manual Lock — Lock session instantly at any time

                            
⚙️ Legacy Fallback — AES-GCM fallback when WebAuthn unavailable

                        

                        


                        
💾 Smart Caching

                        
Visited sites are cached for lightning-fast repeat loads.

                        

                            
🏃‍♂️ Instant Loading — Cached sites load immediately

                            
💽 IndexedDB Storage — Persistent across browser sessions

                            
🧹 Auto Cleanup — Old cache expires after 7 days

                            
🔒 Secure Storage — Only cached after DOMPurify sanitization

                        

                        


                        
🛡️ Security Features

                        
Multi-layer security with DOMPurify integration.

                        

                            
🧼 XSS Protection — All HTML sanitized with DOMPurify

                            

                                🚫 Malicious Code Removal — Dangerous tags and attributes filtered
                            

                            
📦 Sandboxed Execution — Sites run in isolated iframe environment

                            
🔍 Content Validation — All resources validated before display

                        

                    


                    

                        
📋 Requirements for Hosted Sites

                        

                            
📄 Must contain an index.html file in the bundled artifact

                            
🔗 All resources should use relative paths

                            
⚡ Only static content (HTML, CSS, JS, images, fonts, etc.)

                            
🔒 Files are served in a sandboxed environment for security

                            
🧠 Deploy flow seeds from an in-memory bundle, not directly from local folder handles

                            
✅ On load, signature metadata is verified and mirrored into manifest.web25.json

                            
🧷 Signed torrents include publisher + signature metadata for validation

                            
📱 Responsive design recommended for best experience

                            
🧹 Content is automatically sanitized for security
